Coding with Class

09/28/2014

Alright, folks--today, we're going to talk Ruby Classes--and how darn cool they are! We're not going to go too far into this today, but we'll cover enough to get you started.

Hypothetical situation--let's say you want to make a program that will allow the user to enter their DVD collection.

The first thing you should do is make a class!

class DVD
 end

There ya go. Pretty simple. But this doesn't really do much for us. Next up, we've got to give it an initialize function--this will let the user enter new DVDs into the program.

class DVD
 def initialize(title, genre, year)
 @title = title
 @genre = genre
 @year = year
 end
 end

Alright--a bit more is going on here, so let's talk it through. First, we define a method called "intialize". And when the user calls it, they're going to enter three parameters--the title, the genre, and the year. (Obviously we could ask for more information--but I'm trying to keep this simple.)

You're probably wondering what's going on with this "@title = title" bit. A fair question. A variable with preceded by a "@" symbol is an instance variable. Instance variables are useable throughout the class--And we're INITALLY setting each new object's values to whatever was entered by the user.

And now, we have a fully defined class. Still a bit boring, though, right? Right.

The cool thing is that we can ALSO create methods that work on a class. So let's add this--it's super simple, but hey--we're just learning how to crawl, right? Running comes later.

class DVD
 def initialize(title, genre, year)
 @title = title
 @genre = genre
 @year = year
 end
 def title
 p @title
 end
 end

So now, we've made a method called "#title". So if we enter...

filth = DVD.new("Filth", "drama", 2013)

That's going to create a variable called "filth"--and it points to an object (made up of the info in the parentheses) which is a part of the class DVD.

If I then enter...

a.title

Ruby will return...

"Filth"

And this is the starting point of working with Classes in Ruby. You could keep adding methods to this class, work out ways to organize them, etc.

Keep on coding, friends. Keep on coding.
